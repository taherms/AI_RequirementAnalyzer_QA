How to Use

  1. Optionally enter your OpenRouter API key (recommended to avoid rate limiting)
  2. Upload a text file with your software requirements or use the sample file
  3. Click "Analyze Requirements" to generate a summary and test cases
  4. View the results and download the analysis

Requirements File Format

Your requirements file should be a plain text file with clear sections describing:

  • Functional requirements
  • User stories or use cases
  • Technical constraints
  • Non-functional requirements

See the sample_requirements.txt file for an example format.
